# README Demo Script

Use this file as the default capture brief for the README hero GIF and static screenshots.

## Primary GIF

- Target file: `readme-assets/demo.gif`
- Target length: 20 to 30 seconds
- Target size: around 1440px wide before export, optimized for README playback
- Goal: show that `dashboard-LAB` is a local AI workspace, not a recording-only tool

## GIF Flow

1. Open the app in `간단 모드`
2. `Projects` tab: show the local project list and current repo context
3. `Doc Hub` tab: open a project doc to show context lookup
4. `CS Helper` tab: paste a short customer message and show a generated reply draft
5. `Call → PRD` tab: paste a short meeting note and start document generation
6. `Info Hub` tab: show daily updates, trends, or package/security checks
7. End on `Home` or the app header so the product identity is visible

## Static Screenshots

Capture these after the GIF and export them as PNG files:

- `readme-assets/home.png`
- `readme-assets/projects.png`
- `readme-assets/cs-helper.png`
- `readme-assets/call-to-prd.png`

## Capture Rules

- Use clean sample data only. Do not show private customer text, API keys, local usernames, or personal file paths.
- Keep the window size consistent across all shots.
- Prefer `간단 모드` for the GIF unless a specific advanced feature must be shown.
- Keep the pace readable enough that a first-time visitor can understand the flow without pausing.
- Avoid showing setup friction. Start from an already running app.

## Recommended Sample Inputs

### CS Helper

- Customer message: `The export button fails after I upload a CSV. Can you help?`
- Additional context: `Acknowledge the issue, ask for reproduction details, and provide a temporary workaround if possible.`

### Call → PRD

- Input text: `Customer support and PM agreed to add CSV export retry handling, surface a clearer error message, and log failed export jobs for investigation.`
- Project selection: choose a repo that already has docs visible in `Doc Hub`
